# My_Application_11
mobilki

Описание: Мобильное приложение для создания и прохождения тестов, позволяющее управлять тестами и вопросами, делиться ссылкой на прохождение и отслеживать результаты с аналитикой и статистикой.

Инструкция по установке:
1. Скопировать файл  
   - Передать apk на телефон.  
   - Запомнить папку, куда он сохранён (например, Download).

2. Разрешить установку из неизвестных источников  
   - Открыть «Настройки» → «Безопасность» или «Приложения».  
   - Найти пункт «Установка неизвестных приложений» / «Неизвестные источники».  
   - Выдать разрешение для того приложения, через которое будешь открывать apk (например, «Файлы», «Chrome», «Telegram»).

3. Установить apk  
   - Открыть файловый менеджер на телефоне.  
   - Найти apk-файл и нажать на него.  
   - В появившемся окне выбрать «Установить» и дождаться завершения.

4. Запустить приложение  
   - После установки можно нажать «Открыть» сразу.  
   - Либо найти иконку приложения в списке программ и запустить оттуда.

Архитектура базы данных (Firebase Realtime Database)
База данных реализована в виде иерархического JSON-дерева.
Корневая структура БД
Узел user
Хранит данные зарегистрированных пользователей.
user
└── {userId}
├── email
├── fcmToken
├── profileImage
└── testResults
Описание полей:
•	userId — уникальный идентификатор пользователя (UID Firebase)
•	email — электронная почта пользователя
•	fcmToken — токен для push-уведомлений
•	profileImage — ссылка на изображение профиля (Firebase Storage)
•	testResults — результаты пройденных тестов

Узел testResults
testResults
└── {testId}
└── {timestamp}
├── answers
├── duration
├── participantName
├── percentage
├── score
├── totalQuestions

Описание полей:
•	testId — идентификатор теста
•	timestamp — время прохождения теста
•	answers — ответы пользователя
•	duration — время прохождения (в секундах)
•	participantName — имя участника
•	percentage — процент правильных ответов
•	score — количество правильных ответов
•	totalQuestions — общее количество вопросов
Структура ответов (answers)
answers
├── q1 : "a"
├── q2 : "b"
├── q3 : "c"
├── q4 : "a"
└── q5 : "b"
Каждый ключ соответствует номеру вопроса, значение — выбранный вариант ответа.
Узел publicTests
Используется для хранения публичных тестов, доступных всем пользователям приложения.
publicTests
└── {testId}
├── title
├── description
├── questions
└── authorId
